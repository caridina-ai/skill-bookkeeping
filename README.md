# 在 chat room 裡使用的自然語言現金記帳 skill

這個專案把 agent 所服務的 chat room 或 thread 變成現金帳本入口。你可以直接使用 TUI，也可以透過 OpenClaw、Hermes 等 agent 從 Telegram、Discord 或其他 channel 連進同一個記帳對話；因此人在外面時，只要拿手機打開專用的 Telegram chat room 就能記帳。

## 安裝

Clone 或下載本專案後，把 `SKILL.md` 與 `scripts/` 複製到 agent 的 `<SKILLS_DIR>/bookkeeping`。請讓該 agent 能從你選定的 TUI、Telegram、Discord 或其他 channel 收到訊息，並把 `/bookkeeping` 暴露為可載入此 skill 的指令。第一次實際記帳時會自動建立 `data/book.db`。

最簡單的部署方式，是讓一個專用的記帳 chat room 或 thread 對應一份 `bookkeeping` skill 與 `data/`。同一份安裝目錄共用同一本帳；若需要彼此獨立的多本帳，請使用不同的安裝目錄與資料目錄。

## 快速開始：先載入，再逐則記帳

在準備用來記帳的 chat room 或 thread 中，先把下面這一則訊息單獨送出：

```text
/bookkeeping
```

等 agent 載入 skill 後，下一則訊息開始直接輸入記帳內容，不要再加 `/bookkeeping`：

```text
看牙醫 250
```

AI 會回覆：

```text
保健費 (看牙醫) 250 餘額 -250
```

接著可以在同一個對話裡繼續逐則傳送。下面每個區塊都是一則獨立訊息：

```text
國泰世華提款 2000
```

```text
現在餘額多少
```

```text
七月明細
```

`/bookkeeping` 是進入記帳模式的啟動指令，不是每筆帳目的前綴。

## 開箱即用的科目

全新帳本的餘額是 0，並已經有原本 16 個預設科目，不需初始化就能直接記帳：

```text
外食費
買菜金
民俗節日
居住費
管理費
交通費
車稅險
保健費
治裝費
精進金
旅遊金
公關費
孝親費
奉獻
歸墊
雜費
```

輸入「列出所有科目」就能隨時查看目前清單。

## 自訂科目

科目不是寫死的。你可以一次換掉整份清單：

```text
捨棄原本的科目，改用我的
餐飲
買菜
交通
醫療
```

會得到：

```text
已替換科目（共 4 個）
```

也可以說「新增科目 寵物費」、「把寵物費更名為毛孩費」、「刪除科目 毛孩費」或「刪除全部科目」。更名會連同舊帳一起更新；若仍有帳目使用某科目，刪除、清空與整份替換都會拒絕，必須先刪除那些帳目，避免歷史資料失去科目。

若你已自訂科目，AI 只會使用目前存在的名稱。它不應偷偷恢復預設科目，也不應在無法分類時擅自新增科目。

如果你明確刪除全部科目，重新啟動後仍會維持空清單，不會自動補回預設科目；只有刪除整個 `data/book.db`、建立全新帳本時，才會重新建立原本 16 科。

## 匯入舊帳與起始餘額

可以把舊帳分行貼上，一次處理：

```text
一次處理以下紀錄
7/1 10:12 買菜金 183
7/1 12:20 外食費 40
7/1 18:22 外食費 140
7/2 12:15 外食費 170
7/3 18:40 外食費 40
```

接著說「起始餘額 2000」。這表示把匯入舊帳後的目前餘額校正為 2000，不是再入金 2000；以上舊支出共 573，因此此例會回覆：

```text
餘額校正 -573 -> 2000
```

## 平時記帳

以下日常範例是另一個帳本情境，假設輸入前餘額為 12100。一般支出只要說品項與金額：

```text
胃腸藥 180
```

```text
保健費 (胃腸藥) 180 餘額 11920
```

提款、酬勞、中獎、退款或撿到錢會讓手上現金增加：

```text
國泰世華提款 2000
```

```text
入金 (國泰世華提款) 2000 餘額 13920
```

「提款」本身是入金；只有「提款手續費」是支出。

如果有指定日期與時間，確認訊息會完整保留到分鐘：

```text
7/9 14:55 朝代牙科 250
```

```text
2026-07-09 14:55 保健費 (朝代牙科) 250 餘額 13670
```

只寫日期時，確認訊息只顯示日期；完全不寫日期時間時，確認訊息保持簡潔，但帳本仍會記下當下時間。

金額以整數記錄，可以使用 `1,000` 這類千分位寫法，但不支援小數金額；同一本帳也不會自動換算幣別。

## 用餘額倒推支出

採買太多筆來不及逐項輸入時，可以說：

```text
買菜後剩 10250
```

程式會用目前餘額計算這次花費。因為歷史餘額不等於現在餘額，倒推不能搭配「昨天」、特定日期或特定時間；這種情況會請你改提供實際花費，不會硬算。

## 改帳與刪帳

可以說「昨天胃腸藥記錯了，是 280 才對」或「剛剛撿到 200 那筆刪掉」。盡量描述你記得的品項、日期、時間或原金額即可，AI 會自行查找。只有一筆符合時才會動帳；若仍有多筆可能，系統會列出候選紀錄請你確認，不會擅自挑一筆修改或刪除。

## 校正與查詢餘額

「現在剩多少錢？」只查詢餘額。如果你是在告知實際清點結果，則說「我現在實際上有 1883」：

```text
餘額校正 1247 -> 1883
```

校正只把總餘額平移到正確值，不改既有帳目，也不出現在支出報表。

## 明細、分帳與展開

各名稱有固定意義：

| 你可以說 | 顯示內容 |
|---|---|
| 明細、流水、流水帳、依日期、某天花了什麼 | 依日期與 HH:MM 排列的支出 |
| 分帳、分類帳、分類統計、科目統計、支出統計、月報、結算 | 各科目的支出合計 |
| 分帳明細、分類明細、科目明細、依科目明細 | 依科目分組列出每一筆支出 |
| 展開交通費、展開交通費和保健費 | 只列指定科目的分帳明細 |

報表只計支出，不會混入提款、其他入金、餘額校正或封存用期初餘額。

期間與科目都能複選，順序照你說的排列：

```text
六月和七月分帳
7/8 和 7/11 明細
六月和七月流水帳
展開六月和七月的交通費、保健費
展開 7/8 和 7/11 的交通費、保健費
```

明細、分帳、分帳明細與展開都可以一次選多個日期，或一次選多個月份，但同一次不要混用日期與月份。展開也能一次選多個科目。

未指定日期或月份時，報表預設查詢當月。只有日期、沒有時間的補記帳目，會在明細、分帳明細、展開與封存 CSV 中顯示為 `00:00`；同一分鐘有多筆時依建立順序排列。

分帳與分帳明細只列出該期間實際有支出的科目；若整個期間沒有支出，仍會顯示合計 0。使用「展開」明確指定科目時，即使該科目沒有帳目，也會保留科目名稱與合計 0，方便確認查詢範圍。

報表的期間標題線下方會直接接第一個科目、日期或帳目；不同科目、不同日期及不同期間之間各保留一個空行，合計前統一使用 `-------` 分隔，方便快速掃讀。

## 年度封存

說「封存 2026 年的舊帳」，系統會把 2027 年以前的紀錄匯出成 Excel 可直接開啟的 UTF-8-SIG CSV，再從資料庫移除舊帳並補入正確期初餘額。CSV 包含日期、時間、科目、品項、金額與逐筆餘額；封存前後總餘額及自訂科目清單都不會改變。每次封存使用不重複的檔名，而且只有 CSV 成功寫入後才會刪除舊帳；若封存失敗，資料庫與未完成的備份都不會留下半套狀態。

## 資料與更新

帳目保存在安裝目錄的 `data/book.db`，封存檔也放在 `data/`。更新 skill 時只需更換 `SKILL.md` 與 `scripts/`；請保留並定期備份 `data/`。若真的要建立全新帳本，刪除 `data/book.db` 後，下次使用會重新建立原本 16 個預設科目。

`book.db` 與封存 CSV 都是未加密的本機檔案。請依部署環境妥善限制檔案權限、保存備份，並避免把真實帳本提交到公開版本庫；本專案已預設忽略 `data/`。

### 自訂資料庫位置（進階）

一般使用者不需設定。若部署環境需要使用持久化磁碟，或讓不同 chat room 使用不同帳本，可在 agent 的執行環境設定 `BOOK_DB` 為資料庫的絕對路徑。請確認實際執行 `book.py` 的工具程序會繼承此環境變數；未設定時仍使用安裝目錄的 `data/book.db`。
